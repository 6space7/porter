# Security Policy

porter is intended to manage user servers, source code, deployments, domains, and secrets. Security issues should be reported privately before public disclosure.

## Reporting

Until a dedicated disclosure channel is published, please open a private maintainer contact path with the repository owner and include:

- a description of the issue;
- affected versions or commits, if known;
- reproduction steps;
- impact and any suggested mitigations.

## Expectations

- Do not publicly disclose exploitable details before maintainers have had a reasonable chance to respond.
- Do not access, modify, or destroy data that does not belong to you.
- Do not use findings to disrupt services.
